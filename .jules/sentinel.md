## 2025-06-11 - Object Type Confusion Denial of Service in Crypto Operations
**Vulnerability:** Express routing inherently accepts JSON objects in `req.body`. When a user provides an object (e.g., `{"$ne": null}`) instead of a string for the `password` field in `/api/auth/login` or `/api/auth/register`, this object gets passed down to `crypto.pbkdf2Sync(password, salt, ...)` inside `hashPassword` and `verifyPassword`. The native `crypto` module expects a string or buffer and will immediately throw a `TypeError: The "password" argument must be of type string or an instance of ArrayBuffer...`, causing an unhandled promise rejection and crashing the Node.js process (DoS).
**Learning:** Even if a request property does not hit the database directly, passing an unvalidated/uncast object to internal Node.js modules like `crypto` can result in uncaught TypeErrors. Object Type Confusion extends beyond NoSQL Injection into Denial of Service.
**Prevention:** Always explicitly cast unvalidated user request properties (e.g., `req.body`, `req.query`, `req.headers`, `socket.handshake`) to primitives like `String` right at the boundary or before passing them to sensitive functions like database drivers or native libraries (`crypto`). For instance: `const safePassword = String(password || '');`.
